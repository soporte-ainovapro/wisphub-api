## GET /api/v1/clients/search

Performs a flexible search for clients using a general search term. Ideal for finding clients by full name, address, or partial keyword matches when you don't have the document number or phone number.

### Query Parameters

<ParamField query="q" type="string" required>
  Search query string. Can be a partial or full name, address, or any keyword associated with the client. The search is case-insensitive and supports partial matches.
</ParamField>

### Response

Returns a `BackendResponse` containing an array of matching `ClientResponse` objects.

<ResponseField name="ok" type="boolean" required>
  Indicates if the request was successful
</ResponseField>

<ResponseField name="type" type="string" required>
  Response type (e.g., "success", "error")
</ResponseField>

<ResponseField name="action" type="string" required>
  Action identifier: `clients_listed` (when results found) or `client_not_found` (when no results)
</ResponseField>

<ResponseField name="data" type="array" required>
  Array of matching client objects (empty array if no matches found)
  
  <Expandable title="ClientResponse properties">
    <ResponseField name="service_id" type="integer">
      Unique service identifier in WispHub
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Client's full name
    </ResponseField>
    
    <ResponseField name="document" type="string">
      Client's identity document (cédula)
    </ResponseField>
    
    <ResponseField name="phone" type="string">
      Client's phone number
    </ResponseField>
    
    <ResponseField name="address" type="string">
      Client's physical address
    </ResponseField>
    
    <ResponseField name="city" type="string">
      Client's city
    </ResponseField>
    
    <ResponseField name="locality" type="string">
      Client's locality or neighborhood
    </ResponseField>
    
    <ResponseField name="payment_status" type="string">
      Current payment status
    </ResponseField>
    
    <ResponseField name="zone_id" type="integer">
      Service zone identifier
    </ResponseField>
    
    <ResponseField name="antenna_ip" type="string">
      IP address of the assigned antenna
    </ResponseField>
    
    <ResponseField name="cut_off_date" type="string">
      Service cut-off date
    </ResponseField>
    
    <ResponseField name="outstanding_balance" type="float">
      Outstanding balance amount
    </ResponseField>
    
    <ResponseField name="lan_interface" type="string">
      LAN interface identifier
    </ResponseField>
    
    <ResponseField name="internet_plan_name" type="string">
      Name of the internet plan
    </ResponseField>
    
    <ResponseField name="internet_plan_price" type="float">
      Price of the internet plan
    </ResponseField>
    
    <ResponseField name="technician_id" type="integer">
      Assigned technician identifier
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="message" type="string">
  Optional message providing additional context
</ResponseField>

### Example Request

<CodeGroup>
```bash curl
curl -X GET "https://api.wisphub.example/api/v1/clients/search?q=Juan%20Perez" \
  -H "Content-Type: application/json"
```

```python Python
import requests

url = "https://api.wisphub.example/api/v1/clients/search"
params = {"q": "Juan Perez"}
headers = {
    "Content-Type": "application/json"
}

response = requests.get(url, params=params, headers=headers)
print(response.json())
```

```javascript JavaScript
const searchQuery = encodeURIComponent('Juan Perez');

fetch(`https://api.wisphub.example/api/v1/clients/search?q=${searchQuery}`, {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json'
  }
})
.then(response => response.json())
.then(data => console.log(data));
```
</CodeGroup>

### Error Responses

<ResponseField name="400 Bad Request">
  Search query parameter is missing
  
  ```json
  {
    "ok": false,
    "type": "error",
    "action": "error",
    "data": null,
    "message": "Search query parameter 'q' is required"
  }
  ```
</ResponseField>

<Note>
  This endpoint is ideal for implementing search-as-you-type functionality in your application. It supports partial matches and is case-insensitive.
</Note>

<Warning>
  For better performance, it's recommended to use more specific endpoints (by-document, by-phone, by-service-id) when you have exact identifiers available.
</Warning>
