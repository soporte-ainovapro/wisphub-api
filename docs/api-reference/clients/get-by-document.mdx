## GET /api/v1/clients/by-document/{document}

Retrieves client information using their identity document (cédula). Searches the WispHub system and returns the associated service details.

### Path Parameters

<ParamField path="document" type="string" required>
  The client's identity document number (cédula). This should be provided without spaces or special characters.
</ParamField>

### Response

Returns a `BackendResponse` containing a single `ClientResponse` object.

<ResponseField name="ok" type="boolean" required>
  Indicates if the request was successful
</ResponseField>

<ResponseField name="type" type="string" required>
  Response type (e.g., "success", "error")
</ResponseField>

<ResponseField name="action" type="string" required>
  Action identifier: `client_found` or `client_not_found`
</ResponseField>

<ResponseField name="data" type="object">
  Client information object
  
  <Expandable title="ClientResponse properties">
    <ResponseField name="service_id" type="integer">
      Unique service identifier in WispHub
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Client's full name
    </ResponseField>
    
    <ResponseField name="document" type="string">
      Client's identity document (cédula)
    </ResponseField>
    
    <ResponseField name="phone" type="string">
      Client's phone number
    </ResponseField>
    
    <ResponseField name="address" type="string">
      Client's physical address
    </ResponseField>
    
    <ResponseField name="city" type="string">
      Client's city
    </ResponseField>
    
    <ResponseField name="locality" type="string">
      Client's locality or neighborhood
    </ResponseField>
    
    <ResponseField name="payment_status" type="string">
      Current payment status
    </ResponseField>
    
    <ResponseField name="zone_id" type="integer">
      Service zone identifier
    </ResponseField>
    
    <ResponseField name="antenna_ip" type="string">
      IP address of the assigned antenna
    </ResponseField>
    
    <ResponseField name="cut_off_date" type="string">
      Service cut-off date
    </ResponseField>
    
    <ResponseField name="outstanding_balance" type="float">
      Outstanding balance amount
    </ResponseField>
    
    <ResponseField name="lan_interface" type="string">
      LAN interface identifier
    </ResponseField>
    
    <ResponseField name="internet_plan_name" type="string">
      Name of the internet plan
    </ResponseField>
    
    <ResponseField name="internet_plan_price" type="float">
      Price of the internet plan
    </ResponseField>
    
    <ResponseField name="technician_id" type="integer">
      Assigned technician identifier
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="message" type="string">
  Optional message providing additional context
</ResponseField>

### Example Request

<CodeGroup>
```bash curl
curl -X GET "https://api.wisphub.example/api/v1/clients/by-document/1234567890" \
  -H "Content-Type: application/json"
```

```python Python
import requests

url = "https://api.wisphub.example/api/v1/clients/by-document/1234567890"
headers = {
    "Content-Type": "application/json"
}

response = requests.get(url, headers=headers)
print(response.json())
```

```javascript JavaScript
fetch('https://api.wisphub.example/api/v1/clients/by-document/1234567890', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json'
  }
})
.then(response => response.json())
.then(data => console.log(data));
```
</CodeGroup>

### Error Responses

<ResponseField name="404 Not Found">
  No client found with the provided document number
  
  ```json
  {
    "ok": false,
    "type": "error",
    "action": "client_not_found",
    "data": null,
    "message": "Client not found"
  }
  ```
</ResponseField>

<ResponseField name="400 Bad Request">
  Invalid document format
  
  ```json
  {
    "ok": false,
    "type": "error",
    "action": "invalid_document",
    "data": null,
    "message": "Invalid document format"
  }
  ```
</ResponseField>

<Note>
  This is one of the most reliable ways to look up a client, as the identity document is a unique identifier. Ensure the document number is provided without spaces or special characters.
</Note>
